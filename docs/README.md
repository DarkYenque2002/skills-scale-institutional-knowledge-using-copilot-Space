# OctoAcme Project Management Processes

Welcome to the OctoAcme project management documentation. This folder contains comprehensive guides for running projects, managing teams, and delivering value consistently across our organization.

## Overview

OctoAcme follows a structured, iterative approach to project management grounded in customer value, clear ownership, data-informed decisions, iterative delivery, and psychological safety.

## Process Documentation

The OctoAcme project lifecycle is divided into five key phases:

1. [Project Initiation](./octoacme-project-initiation.md)
2. [Project Planning](./octoacme-project-planning.md)
3. [Execution & Tracking](./octoacme-execution-and-tracking.md)
4. [Release & Deployment](./octoacme-release-and-deployment.md)
5. [Retrospective & Continuous Improvement](./octoacme-retrospective-and-continuous-improvement.md)

## Roles and Personas

The main roles involved in OctoAcme project management are:

- **Project Manager**: Coordinates delivery, tracks timelines, manages risks, and keeps stakeholders aligned.
- **Product Manager**: Defines project goals, prioritizes work, and measures product outcomes.
- **Developers**: Design, build, test, and deliver software components.
- **QA / Testing**: Validates quality, acceptance criteria, and release readiness.
- **Stakeholders**: Provide business input, feedback, approvals, and project direction.

## Key Artifacts

Every OctoAcme project maintains:

- Project charter or one-pager
- Roadmap and release plan
- Sprint or iteration backlog
- Acceptance criteria and Definition of Done
- Risk register
- Retrospective notes

## Communication Cadence

- Weekly sync between Project Manager and Product Manager
- Twice-weekly standups for the delivery team
- Monthly stakeholder updates
- Ad-hoc escalations for blockers or risks

## Getting Started

1. New project? Start with [Project Initiation](./octoacme-project-initiation.md).
2. Approved and ready to plan? Move to [Project Planning](./octoacme-project-planning.md).
3. In active delivery? Use [Execution & Tracking](./octoacme-execution-and-tracking.md).
4. Approaching release? Review [Release & Deployment](./octoacme-release-and-deployment.md).
5. Project complete? Conduct a [Retrospective](./octoacme-retrospective-and-continuous-improvement.md).
